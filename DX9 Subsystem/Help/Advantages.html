Advantages:

  • DisplaySprite(), DisplayTransparentSprite() works faster, if a newer gfx-card is available.

  • DisplayTranslucentSprite(),DisplayRGBFilter(),DisplayAlphaSprite(),DisplayShadowSprite() and DisplaySolidSprite() are hardware accelerated and much faster.

  • All sprites are backed in system memory, so sprites can be restored, if the device gets lost.

  • SpritePixelCollision() works much faster for static sprites.

  • 3D-Sprites are sometimes faster.

Disadvantages:

  • Direct3D9 and a 3d accelerator are needed.

  • The subsystem needs much more memory.

  • UseBuffer(),TransparentSpriteColor(),SpriteOutput() can be very slow if PS 1.4 is not available.

  • CopySprite(),GrabSprite() are really slow.

  • sprites are limited to the maximal texture size. (usually 2048x2048)
