Differences between the DX7- and the DX9 Rendersystem:

 

OpenScreen():
The OpenScreen()-command only supports resoultions of 16Bit and 32Bit. 8Bit and 24Bit resolutions are not supported. Use the DX7 Rendersystem if you want to support them too.

OpenWindowedScreen():
The OpenWindowedScreen()-command only supports desktop resoultions of 16Bit and 32Bit. 8Bit and 24Bit resolutions are not supported. Use the DX7 Rendersystem if you want to support them too.

StartSpecialFX():
StartSpecialFX() is only available for compatibility reason. It does nothing.

StopSpecialFX():
StopSpecialFX() is only available for compatibility reason. It does nothing.

DisplayRGBFilter():
The command DisplayRGBFilter() is now hardware accelerated. So it's much faster.

FlipBuffers():
FlipBuffers(2) works also in windowed mode.

DisplayTranslucentSprite():
The transparent color set With TransparentSpriteColor() is used as transparent color instead of always black.
This command is now hardware accelerated. So it's much faster.

UseBuffer():
Try to avoid to use UseBuffer(Sprite) in the main loop whenever it's possible. It's not recommented to use this command in the main loop, because it's really slow. If you need to use UseBuffer() often, you should use the DX7 Rendersystem!

TransparentSpriteColor():
Try to avoid to use TransparentSpriteColor() in the main loop.If you need to use TransparentSpriteColor() often in the main loop, you should use the DX7 Rendersystem!

ChangeGamma():
This command does nothing for now.

SpritePixelCollision():
Works the same way like SpriteCollision() for now.

GrabSprite():

GrabSprite creates only a new sprite when it's needed.(e.g. another size or another bit depth)
Try to avoid to use GrabSprite() in the main loop whenever it's possible. It's not recommented to use this command in the main loop, because it's really slow. If you need to use GrabSprite() often, you should use the DX7 Rendersystem!

CopySprite():

CopySprite creates only a new sprite when it's needed.(e.g. another bit depth)
Try to avoid to use CopySprite() in the main loop whenever it's possible. It's not recommented to use this command in the main loop, because it's really slow. If you need to use GrabSprite() often, you should use the DX7 Rendersystem!

SetFrameRate():

FrameRate is set to 0 by default. (0 works, in contrast to the DX7 Rendersystem)

CreateSprite():

In contrast to the DX7 Rendersystem CreateSprite() creates always a texture ressource. This means, that the size of a sprite is limited to the maximum size of a texture.(mostly 2048x2048)

The alphachannel of textures created with #PB_Sprite_AlphaBlending is inverted.(This makes it possible to draw correct on alphachannel textures with SpriteOutput())
