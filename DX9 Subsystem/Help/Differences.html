Differences between the DX7- and the DX9 render system:

There are a few differences to the DX7 render system. Some of them are there because it's not possible to do it the same way like in the DX7 render system, others are different for speed or quality improvement.

OpenScreen():
The OpenScreen()-command only supports resoultions of 16Bit and 32Bit. 8Bit and 24Bit resolutions are not supported. Use the DX7 render system if you want to support them too.

OpenWindowedScreen():
The OpenWindowedScreen()-command only supports desktop resoultions of 16Bit and 32Bit. 8Bit and 24Bit resolutions are not supported. Use the DX7 render system if you want to support them too.

StartSpecialFX():
StartSpecialFX() is only available for compatibility reason. It does nothing.

StopSpecialFX():
StopSpecialFX() is only available for compatibility reason. It does nothing.

DisplayRGBFilter():
The command DisplayRGBFilter() is hardware accelerated. So it's much faster.

FlipBuffers():
FlipBuffers(2) works also in windowed mode.

DisplayTranslucentSprite():
The transparent color set with TransparentSpriteColor() is used as transparent color instead of always black.
This command is hardware accelerated. So it's much faster.

UseBuffer():
You can't display a sprite while it's used as render target.

UseBuffer(1) ; use sprite as render target
DisplaySprite(1,x,y) ; doesn't work!

It's possible to use DisplaySprite3D() with UseBuffer(), that means you can display 3D-Sprites
on a sprite.

Try to avoid to use UseBuffer(Sprite) in the main loop whenever it's possible. It's not recommended to use this command in the main loop, because it's really slow. If you need to use UseBuffer() often, you should use the DX7 render system!
However, if there is at least PS 1.4 support, then this command works fairly fast.

TransparentSpriteColor():
Try to avoid to use TransparentSpriteColor() in the main loop.If you need to use TransparentSpriteColor() often in the main loop, you should use the DX7 render system!
However, if there is at least PS 1.4 support, then this command works really fast.

GrabSprite():
GrabSprite creates only a new sprite when it's needed.(e.g. another size or another bit depth)
Try to avoid to use GrabSprite() in the main loop whenever it's possible. It's not recommended to use this command in the main loop, because it's really slow. If you need to use GrabSprite() often, you should use the DX7 render system!

CopySprite():
CopySprite creates only a new sprite when it's needed.(e.g. another bit depth)
Try to avoid to use CopySprite() in the main loop whenever it's possible. It's not recommended to use this command in the main loop, because it's really slow. If you need to use CopySprite() often, you should use the DX7 render system!

SetFrameRate():
If you set FrameRate to 0, FlipBuffers() will not wait, except for the vertical-blank interval.

DisplayAlphaSprite():
For now this command doesn't work correctly. This command is hardware accelerated. So it's much faster.

DisplayShadowSprite():
This command is hardware accelerated. So it's much faster.

DisplaySolidSprite():
This command is hardware accelerated. So it's much faster.
