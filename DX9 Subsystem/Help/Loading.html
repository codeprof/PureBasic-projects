How to improve loading speed of LoadSprite() and CatchSprite()?

You can easyly reduce the needed time for loading if you load your sprite with the flags #PB_Sprite_Memory or #PB_Sprite_Texture|#PB_Sprite_AlphaBlending.
If you don't want to use these types of sprite (as #PB_Sprite_Memory is slower in displaying) you should avoid TransparentSpriteColor() calls after loading the sprite, as they are really really slow:

Faster way:

TransparentSpriteColor(#PB_Default,#Magenta)
LoadSprite(1,“sprite1.bmp“)
LoadSprite(2,“sprite2.bmp“)
LoadSprite(3,“sprite3.bmp“)
...



Slow way:

LoadSprite(1,“sprite1.bmp“)
TransparentSpriteColor(1,#Magenta)
LoadSprite(2,“sprite2.bmp“)
TransparentSpriteColor(2,#Magenta)
LoadSprite(3,“sprite3.bmp“)
TransparentSpriteColor(3,#Magenta)
...


Note:
Even if you use this faster way, loading sprites with the dx9 subsystem is slower than with the dx7 subsystem, because there is much more internal work to do.
